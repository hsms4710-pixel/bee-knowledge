## GDD MCP Server 配置

### 问题发现
之前的开发中，我直接手写了代码和图谱，没有调用 GDD 的 MCP 工具。原因是：
1. GDD MCP Server 有依赖问题无法启动
2. MCP 工具未注册到 WorkBuddy

### 修复步骤
1. **修复依赖问题** - `rm -rf node_modules/.bin && npm rebuild`
2. **测试 MCP Server** - `npx tsx src/mcp/server.ts` ✅ 成功启动
3. **配置 WorkBuddy MCP** - 在 `~/.workbuddy/mcp.json` 添加 GDD 配置

### GDD MCP 配置 (更新)
```json
{
  "gdd": {
    "command": "/Users/jiangqiyuan/.workbuddy/binaries/node/versions/22.12.0/bin/npx",
    "args": ["tsx", "src/mcp/server.ts"],
    "cwd": "/Users/jiangqiyuan/Desktop/graph-driven-development",
    "env": {
      "NODE_PATH": "/Users/jiangqiyuan/Desktop/graph-driven-development/node_modules"
    },
    "timeout": 300000,
    "disabled": false
  }
}
```

### 修复的问题
1. 使用完整路径的 npx 命令
2. 添加 NODE_PATH 环境变量
3. 简化 args 参数 (移除绝对路径)

### 最终解决方案
创建了 `src/mcp/server-simple.ts` - 一个简化版的 HTTP MCP Server：
- 不依赖 better-sqlite3（避免原生模块问题）
- 使用文件存储（JSON）
- 支持 MCP 协议和直接 HTTP 调用
- 端口: 3100

配置:
```json
{
  "gdd": {
    "url": "http://localhost:3100/mcp",
    "timeout": 300000,
    "disabled": false
  }
}
```

启动命令:
```bash
cd /Users/jiangqiyuan/Desktop/graph-driven-development
NODE_PATH=node_modules npx tsx src/mcp/server-simple.ts
```

测试结果:
- ✅ 健康检查: `curl --noproxy '*' -s http://localhost:3100/health`
- ✅ 创建图谱: `gdd_create_graph` 成功
- ✅ 工具调用: 返回正确结果
