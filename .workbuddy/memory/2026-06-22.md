## 设计并实现项目知识沉淀系统

**任务内容：**
基于《洛克王国》AI 增效实践，为 graph-driven-development 项目设计并实现了一套完整的知识沉淀系统。

**产出文件：**
1. `docs/项目知识沉淀方案.md` - 完整的系统设计文档
2. `scripts/build-knowledge.ts` - 知识库构建 CLI 工具
3. `docs/知识库使用指南.md` - AI 助手使用指南
4. `.workbuddy/memory/MEMORY.md` - 项目核心信息

**系统架构（MapReduce）：**
```
源代码 → Map 阶段 → Merge 阶段 → Reduce 阶段 → 知识库
         ↓              ↓              ↓            ↓
    提取文件结构    模块内合并    生成模块文档    Markdown/JSON
```

**核心功能：**
1. **Map 阶段**：提取导出函数、类、类型、常量、依赖关系、代码复杂度
2. **Merge 阶段**：按模块合并文件信息，分析模块级依赖
3. **Reduce 阶段**：生成模块 Markdown 文档和项目索引
4. **增量更新**：Hash 缓存检测变更，只处理修改的文件

**知识库输出：**
- `knowledge-base/index.json` - 项目索引（20 个模块）
- `knowledge-base/architecture.md` - 架构总览
- `knowledge-base/modules/*.md` - 20 个模块文档

**CLI 命令：**
```bash
npx ts-node scripts/build-knowledge.ts --full        # 完整构建
npx ts-node scripts/build-knowledge.ts --incremental # 增量更新
npx ts-node scripts/build-knowledge.ts --status      # 查看状态
npx ts-node scripts/build-knowledge.ts --clean       # 清除缓存
```

**统计数据：**
- 处理文件：70 个
- 生成模块：20 个
- 总导出数：200+

**分层检索策略：**
1. 第一层：项目概览（MEMORY.md + index.json）
2. 第二层：架构文档（architecture.md）
3. 第三层：模块详情（modules/*.md）
4. 第四层：源代码（src/*.ts）

**与 WorkBuddy 集成：**
- `.workbuddy/memory/MEMORY.md` 存储项目核心信息
- `knowledge-base/` 存储代码结构知识
- AI 助手按需分层读取，避免大海捞针
