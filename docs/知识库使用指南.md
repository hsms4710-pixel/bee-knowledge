# 知识库使用指南

## 概述

本项目实现了基于《洛克王国：世界》AI 增效实践的**项目知识沉淀系统**，采用 MapReduce 架构从源代码中提取结构化知识。

## 快速开始

### 1. 首次构建（完整构建）

```bash
npx ts-node scripts/build-knowledge.ts --full
```

这将：
- 扫描 `src/` 目录的所有 TypeScript 文件
- 提取类名、函数、接口、枚举等信息
- 合并生成模块级文档
- 处理 `test/` 目录的测试用例
- 输出到 `knowledge-base/` 目录

### 2. 增量更新（推荐日常使用）

```bash
npx ts-node scripts/build-knowledge.ts --incremental
```

这将：
- 检测文件变更（通过 Hash 比较）
- 只处理变更的文件
- 保留未变更的模块文档

### 3. 其他命令

```bash
# 查看知识库状态
npx ts-node scripts/build-knowledge.ts --status

# 清除缓存（重新完整构建前建议执行）
npx ts-node scripts/build-knowledge.ts --clean

# 运行回测评估
npx ts-node scripts/build-knowledge.ts --retro
```

## 知识库结构

构建完成后，`knowledge-base/` 目录结构如下：

```
knowledge-base/
├── index.json           # 项目索引（AI 首先读取）
├── architecture.md      # 架构总览
├── tests.md             # 测试用例知识库
├── retrospective.json   # 回测数据
└── modules/             # 模块文档目录
    ├── core.md          # 核心模块
    ├── mcp.md           # MCP 模块
    ├── editor.md        # 编辑器模块
    └── ...              # 其他模块
```

## AI 助手使用流程

当 AI 助手需要理解项目时，按以下顺序读取：

### 第一步：读取项目记忆

```markdown
.workbuddy/memory/MEMORY.md
```

获取项目概述、技术栈、核心模块等基本信息。

### 第二步：读取知识库索引

```markdown
knowledge-base/index.json
```

获取所有模块列表、统计数据、文件路径。

### 第三步：按需读取模块文档

```markdown
knowledge-base/modules/<module-name>.md
```

根据任务需要，读取相关模块的详细文档。

### 第四步：精准读取源代码

```typescript
src/<module>/<file>.ts
```

只在需要实现细节时读取具体源代码。

## MapReduce 架构说明

### Map 阶段（提取）

从每个文件中提取：

| 提取项 | 说明 |
|--------|------|
| 类名、类功能 | 导出的 class 及其 JSDoc 描述 |
| 函数主要功能 | 导出的 function/method |
| 涉及的模块 | import 的内部依赖 |
| 接口/类型 | 导出的 interface/type |
| 枚举类型 | 导出的 enum |
| 核心流程 | 导出的公共 API 分类 |

### Merge 阶段（合并）

分层合并策略：

```
文件 → 子模块（同目录）→ 模块（同功能域）→ 架构总览
```

- **去重**：相同的类/函数只保留一份
- **关联**：标注引用关系
- **整理**：按功能分组

### Reduce 阶段（生成）

输出高密度的 Markdown 文档：

- `architecture.md` - 架构总览
- `modules/*.md` - 模块文档
- `index.json` - 索引文件

## Hash 缓存机制

系统使用 SHA256 Hash 检测文件变更：

- **文件级 Hash**：单文件变化检测
- **模块级 Hash**：文件集合变化检测
- **测试级 Hash**：测试文件变化检测

缓存存储在 `.knowledge-cache/` 目录。

## 评估体系

### 标准问答集

`knowledge-base/evaluation.json` 包含 20 个标准问答，用于验证知识库效果：

- 架构类问题（4 个）
- API 类问题（6 个）
- 模块关系类问题（4 个）
- 技术细节类问题（4 个）
- 测试类问题（2 个）

### 回测机制

`knowledge-base/retrospective.json` 用于回测评估：

- 根因吻合度
- 模块定位准确性
- 日志解释合理性
- 代码修复适用性

运行回测：

```bash
npx ts-node scripts/build-knowledge.ts --retro
```

## CI/CD 自动化

项目提供了 GitHub Actions 配置：

```
.github/workflows/knowledge-base.yml
```

功能：
- 推送 `src/` 变更时自动增量更新
- 每天凌晨 2 点定时完整重建
- 支持手动触发完整重建

启用方法：
1. 确保 GitHub 仓库启用了 Actions
2. 推送代码后自动执行

## 常见问题

### Q: 增量更新后统计数据变成 0？

A: 这是正常现象。增量模式下，如果文件无变更，会保留模块文档但重新统计索引。实际模块文档内容是完整的。

### Q: 如何强制完整重建？

A: 

```bash
npx ts-node scripts/build-knowledge.ts --clean
npx ts-node scripts/build-knowledge.ts --full
```

### Q: 知识库生成的文档质量如何？

A: 文档基于代码的 JSDoc 注释生成。建议：
- 为导出的函数/类添加 JSDoc 注释
- 使用 `@param`、`@returns` 描述参数和返回值
- 使用 `@example` 提供使用示例
