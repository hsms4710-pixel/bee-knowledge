# 项目知识沉淀系统 - 架构设计（对齐洛克王国）

> 基于《洛克王国：世界》AI 增效实践的原始设计

---

## 一、核心设计理念

### 1.1 核心认知

> "AI 目前的价值还不在于一上来就可以独立投入工作。它只是一个拥有高智商、低记忆力的人。只是这样的人有无数个。"

> "项目级知识沉淀的核心，不是让 AI 记住一切，而是给它一个**稳定入口**，让它在解决问题前先读取项目事实。"

### 1.2 为什么需要知识沉淀

- 小项目：AI 像在泳池中游泳，很快找到内容
- 大项目：AI 像在大海中游泳，需要人为指定方向
- 1M 上下文会被大型项目撑爆

### 1.3 知识沉淀的内容选择

| 高价值（值得沉淀） | 低价值（不值得沉淀） |
|-------------------|---------------------|
| **代码** - 实际运行的最终产物 | **Wiki 文档** - 覆盖面小，时效性差 |
| **测试用例** - 可复用的验证资产 | **策划案** - 格式过多，与实际有偏差 |
| | **TAPD** - 一次性产物，用完即弃 |

## 二、MapReduce 架构

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         MapReduce 知识沉淀架构                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  【输入】源代码                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  src/                                                            │   │
│  │  ├── core/                                                       │   │
│  │  │   ├── graph.ts                                                │   │
│  │  │   ├── node.ts                                                 │   │
│  │  │   └── edge.ts                                                 │   │
│  │  ├── mcp/                                                        │   │
│  │  │   ├── MCPServer.ts                                            │   │
│  │  │   ├── ContextTools.ts                                         │   │
│  │  │   └── ...                                                     │   │
│  │  └── ...                                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  【Map 阶段】单文件分析（最小模块子集）                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  每个文件独立分析，提取结构化信息                                    │   │
│  │  ├── 类名、类功能                                                   │   │
│  │  ├── 函数主要功能                                                   │   │
│  │  ├── 涉及的模块（依赖）                                             │   │
│  │  ├── 接口/类型定义（对应"协议"）                                     │   │
│  │  ├── 枚举类型                                                       │   │
│  │  └── 核心流程（导出 API）                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  【Merge 阶段】模块内合并（分层 Merge）                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  同一目录下的所有文件 JSON → 合并为模块级文档                       │   │
│  │  ├── 去重（相同的类/函数只保留一份）                                │   │
│  │  ├── 关联（标注引用关系）                                          │   │
│  │  └── 整理（按功能分组）                                            │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  【Reduce 阶段】跨模块汇总                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  所有模块文档 → 生成知识库最终文档（架构总览 + 模块索引）             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  【输出】知识库                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  knowledge-base/                                                 │   │
│  │  ├── architecture.md    # 架构总览（Reduce 阶段输出）              │   │
│  │  ├── modules/           # 模块文档（Merge 阶段输出）              │   │
│  │  │   ├── core.md                                                    │   │
│  │  │   ├── mcp.md                                                     │   │
│  │  │   └── ...                                                        │   │
│  │  └── index.json       # 索引文件                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.2 Map 阶段详细设计

**原文描述：**
> "自定义选择需要沉淀的模块子集，子集中的每个文件先进行一次 Map 分析，提炼出这个文件中对于自己业务有意义的东西。"

**TypeScript 项目适配：**

```json
// Map 阶段输出格式（对齐原文）
{
  "file": "src/core/graph.ts",
  "module": "core",
  "classes": [
    {
      "name": "Graph",
      "function": "图数据结构的核心类，管理节点和边",
      "methods": ["addNode", "addEdge", "removeNode", "getNodes", "getEdges"],
      "interfaces": ["GraphNode", "GraphData"],
      "enums": [],
      "coreFlows": {
        "nodeOperations": ["addNode", "removeNode", "updateNode"],
        "edgeOperations": ["addEdge", "removeEdge"],
        "queries": ["getNodes", "getEdges", "getNodeById"]
      }
    }
  ],
  "exports": ["Graph", "GraphNode", "GraphData"],
  "dependencies": {
    "internal": ["./node.ts", "./edge.ts"],
    "external": []
  }
}
```

**Map 提取内容（严格对齐原文）：**

| 原文提取项 | TypeScript 对应 | 说明 |
|-----------|-----------------|------|
| 类名、类功能 | `classes[].name`, `classes[].function` | 导出的 class |
| 函数主要功能 | `classes[].methods` | 导出的 function/method |
| 涉及的模块 | `dependencies.internal` | import 的项目内模块 |
| 协议 | `interfaces` | 导出的 interface/type |
| 枚举类型 | `enums` | 导出的 enum |
| 核心流程 | `coreFlows` | 导出的公共 API 分类 |

### 2.3 Merge 阶段详细设计

**原文描述：**
> "信息沉淀的最小子集应该是最小的模块子集。"
> "再将同一模块下的信息结构向上 Merge 得到模块级别的文档。"
> "分层 Merge，解决信息密度丢失。"

**Merge 策略：**

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Merge 阶段流程（对齐原文）                                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Step 1: 模块内 Merge（小粒度）                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  core/ 模块：                                                    │   │
│  │  ├── graph.ts.json                                              │   │
│  │  ├── node.ts.json                                               │   │
│  │  ├── edge.ts.json                                               │   │
│  │  └── serializer.ts.json                                         │   │
│  │                              ↓ Merge                            │   │
│  │  └── core.md (模块级文档)                                        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Step 2: Merge 时的信息整合                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 去重：相同的类/函数只保留一份                                   │   │
│  │  • 关联：标注类之间的引用关系                                      │   │
│  │  • 整理：按功能分组（数据结构、工具函数等）                         │   │
│  │  • 合并：合并所有文件的依赖关系                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 2.4 Reduce 阶段详细设计

**原文描述：**
> "模块 A 文档 + 模块 B 文档 + ... → Reduce → 知识库最终文档"

**Reduce 输出：**

```
knowledge-base/
├── architecture.md    # 架构总览（Reduce 阶段输出）
├── modules/           # 模块文档（Merge 阶段输出）
│   ├── core.md        # 核心模块
│   ├── mcp.md         # MCP 工具模块
│   ├── components.md  # UI 组件模块
│   └── ...
└── index.json         # 索引文件
```

## 三、Hash 缓存与增量更新

### 3.1 原文描述

> "文件级/模块级 Hash 缓存"
> "每日自动更新沉淀出来的结果"

### 3.2 实现设计

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Hash 缓存机制（对齐原文）                                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  【文件级 Hash】                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  src/core/graph.ts → SHA256 → a1b2c3d4...                       │   │
│  │  src/core/node.ts  → SHA256 → e5f6g7h8...                       │   │
│  │  src/mcp/xxx.ts    → SHA256 → i9j0k1l2...                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  【变更检测】                                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • Hash 未变化 → 跳过处理                                         │   │
│  │  • Hash 已变化 → 重新 Map + Merge                                 │   │
│  │  • 文件新增 → 新增 Map + Merge                                     │   │
│  │  • 文件删除 → 移除对应记录                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              ↓                                          │
│  【模块级 Hash】                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  模块内任一文件变化 → 模块级 Hash 变化 → 重新 Merge               │   │
│  │  模块级 Hash 未变化 → 跳过 Merge                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
